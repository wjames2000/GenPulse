# AI 软件项目开发流水线（桌面版）UX/UI 设计规范

基于《项目需求说明书 V4.0（Go 语言版）》


## 1. 引言

### 1.1 文档目的

本文档旨在定义 AI 软件项目开发流水线（桌面版）产品的用户体验与界面视觉设计规范，涵盖设计原则、界面布局、交互模式、组件规范、色彩系统、字体排版、图标规范及可访问性标准。文档为产品设计师、前端开发工程师及质量保障团队提供统一的视觉语言与交互指导，确保产品具备一致、专业且高效的视觉体验。

### 1.2 适用范围

本规范适用于 AI 软件项目开发流水线桌面应用的所有界面模块，包括但不限于：项目配置界面、Agent 监控仪表盘、技能管理界面、记忆查看器、终端面板及设置界面。

### 1.3 参考文档

- 《项目需求说明书 V4.0（Go 语言版）》
- 《概要设计说明书 V1.0》
- 《详细设计说明书 V1.0》

### 1.4 术语说明

| 术语                            | 说明                                                         |
| ------------------------------- | ------------------------------------------------------------ |
| Design Token                    | 设计令牌，存储视觉属性（颜色、字体、间距等）的键值对，支持跨平台一致应用 |
| Liquid Glass                    | 液态玻璃效果，一种半透明毛玻璃视觉样式，用于浮层组件         |

| Thought Bubble                  | 思维气泡，展示 AI Agent 实时推理过程的对话式 UI 组件         |
| Generative UI                   | 生成式 UI，由 AI 根据上下文动态生成的界面组件                |


## 2. 产品定位与设计哲学

### 2.1 产品定位

**AI 软件项目开发流水线**是一款面向开发者的桌面端 AI 协作平台。区别于传统代码编辑器，本产品核心理念是 **“以 Agent 为中心”** ——用户通过监控和管理一个由多个 AI Agent 组成的虚拟开发团队，来完成从需求到可运行项目的全流程开发。界面设计的首要目标是让用户清晰感知各 Agent 的状态、行为和协作关系，同时保持对代码和文件系统的最终控制权。

### 2.2 设计核心原则

**2.2.1 以 Agent 为中心**

界面布局和交互设计围绕 AI Agent 团队展开。用户无需手动管理文件树，而是通过 Agent 状态面板、执行时间线和思维链面板来了解开发进度。当前 AI 编程工具正经历从“以文件为中心”向“以智能体为中心”的转变，本产品在设计上顺应这一趋势，将 Agent 协作状态作为主视觉核心。

**2.2.2 清晰至上**

AI 开发流程本质上是复杂且非完全确定的。界面的核心任务是通过结构化布局和实时可视化降低认知负担。每个 Agent 的状态（空闲/运行/等待/完成/错误）、思考过程、工具调用结果均需在界面上以高度可读的方式呈现。

**2.2.3 效率优先**

面向专业开发者，交互设计应强调键盘快捷操作、信息密度适中、关键操作路径最短化。支持命令面板（⌘K）快速唤起常用功能。

**2.2.4 透明的可控感**

用户应当始终能够感知 AI 在做什么、为何如此做，并能够随时干预或接管。文件变更以 Diff 形式呈现，命令执行前提供预览，重大操作需用户确认。这一理念与学术研究中“用户始终能感知 AI 行为并随时干预”的可视化设计原则一致。

**2.2.5 沉浸式开发体验**

采用深色模式作为默认主题，减少长时间编码的视觉疲劳。使用微动效传达状态变化，但不干扰核心注意力流。通过统一的 Design Token 管理深浅色模式，确保主题切换的一致性。

## 4. 桌面应用布局规范

### 4.1 窗口尺寸与响应式断点

桌面应用采用 Wails 框架构建，窗口尺寸规范如下：

| 断点类型     | 最小尺寸 | 推荐尺寸 | 最大展开 |
| ------------ | -------- | -------- | -------- |
| 默认窗口     | 1280×800 | 1440×900 | 全屏     |
| 展开面板模式 | 1024×768 | 1280×800 | —        |
| 最小支持     | 1024×640 | —        | —        |

### 4.2 网格系统

采用 8px 基础网格系统，所有间距、内边距、组件尺寸均为 8 的倍数：

| 间距级别 | 数值 | 使用场景                 |
| -------- | ---- | ------------------------ |
| xs       | 4px  | 图标与文字紧贴、紧凑分组 |
| sm       | 8px  | 组件内部元素间距         |
| md       | 16px | 组件之间、卡片内边距     |
| lg       | 24px | 面板之间、内容区块分隔   |
| xl       | 32px | 页面主边距               |
| 2xl      | 48px | 章节分隔                 |

### 4.3 布局模板

**4.3.1 标准三栏布局**

侧边栏（可折叠） + 内容主区 + 右侧辅助面板（可收起），这是应用最核心的布局模式。

```
┌─────────────────────────────────────────────────────────────────┐
│  [Logo]  项目: MyProject ▼    [运行中] ●           [设置] [用户] │  ← 顶部工具栏（56px）
├──────────┬────────────────────────────────────────┬─────────────┤
│          │                                        │             │
│  项目列表 │              主内容区                   │  辅助面板    │
│  (260px) │          (自适应填充)                   │  (360px)    │
│          │                                        │             │
│  ├ 项目A  │  ┌─────────────────────────────────┐  │  Agent 状态 │
│  ├ 项目B  │  │        Agent 状态仪表盘          │  │  ────────  │
│  ├ 项目C  │  └─────────────────────────────────┘  │  当前思维链 │
│          │                                        │             │
│  ┌─────┐ │  ┌─────────────────────────────────┐  │  文件变更   │
│  │+新建│ │  │         执行时间线               │  │  预览      │
│  └─────┘ │  └─────────────────────────────────┘  │             │
│          │                                        │             │
│  执行历史 │  ┌─────────────────────────────────┐  │             │
│  · 今天  │  │         Tab 切换面板              │  │             │
│  · 昨天  │  │  [日志] [终端] [输出文件]        │  │             │
│          │  └─────────────────────────────────┘  │             │
└──────────┴────────────────────────────────────────┴─────────────┘
```

**4.3.2 侧边栏规范**

- 默认宽度：260px
- 可折叠至 48px（仅显示图标）
- 背景色：中性色背景（略深于内容区），形成视觉层级
- 项目列表支持拖拽排序
- 底部固定放置「新建项目」按钮

**4.3.3 辅助面板规范**

- 默认宽度：360px
- 可手动拖拽调整宽度（280px–480px）
- 支持一键收起至右侧边缘
- 内部采用 Tab 或折叠分组组织内容
- 运行状态下，Agent 状态区自动滚动至当前活跃项

**4.3.4 灵活布局**

借鉴 Claude Code 桌面版的设计理念，所有面板均支持拖拽调整位置与尺寸，用户可自由组合工作区布局。


## 5. 页面详细设计

### 5.1 仪表盘页面

**功能定位**：流水线执行的核心监控界面，提供 Agent 团队状态的全局视图。

**5.1.1 Agent 状态仪表盘（顶部核心区）**

以卡片网格形式展示所有参与当前项目的 Agent。每张卡片包含：

- **Agent 头像/图标**：通过色彩与图标区分角色（产品经理、架构师、前端、后端等）
- **Agent 名称与角色**：如“Frontend Dev · React”
- **状态指示灯**：
  - ⚪ 灰色：空闲待命
  - 🔵 蓝色脉冲：运行中
  - 🟡 黄色旋转：等待工具返回
  - 🟢 绿色：已完成
  - 🔴 红色：出错
- **当前任务简述**：如“正在生成 Login 组件...”
- **进度环**：针对运行中的 Agent 显示任务进度百分比
- **Token 消耗**：小字显示已消耗 Token 数

**交互**：点击卡片展开该 Agent 的详细活动面板（含历史工具调用列表）。

**5.1.2 执行时间线**

水平甘特图形式，展示各 Agent 的执行时间段：

- X 轴为时间（相对或绝对），Y 轴为 Agent 角色
- 每个 Agent 的执行块用其代表色标识，长度表示耗时
- 并行执行的 Agent 在同一垂直区域内显示为重叠块
- 悬停显示详细信息（开始时间、结束时间、耗时、状态）
- 支持点击时间块跳转到对应时刻的日志

**5.1.3 Tab 切换面板**

位于主内容区下半部分，包含三个 Tab：

- **日志 Tab**：实时流式展示 Agent 与系统的完整输出日志，支持按级别（INFO/WARN/ERROR）过滤，支持关键词搜索
- **终端 Tab**：内嵌终端模拟器，展示 `shell_exec` 等命令的实时输出
- **输出文件 Tab**：以树形目录展示生成的项目文件结构，点击文件可预览内容

### 5.2 辅助面板组件

**5.2.1 思维链面板**

以对话气泡形式实时展示当前活跃 Agent 的推理过程（Thought），包含以下视觉特征：

- 气泡区分 Agent（不同颜色边框/背景）
- 流式渲染：文字逐字出现，营造“正在思考”的感知
- 支持 Markdown 格式（代码块高亮、列表等）
- 用户可在面板底部输入框直接回复或追问当前 Agent
- 参考 Claude Code 的 Verbose/Normal/Summary 三档显示模式，支持用户自行调节信息密度

**5.2.2 工具调用日志**

以时间倒序列表形式展示：

- 工具名称（如 `fs_write`）+ 状态图标（✓ 成功 / ✗ 失败 / ⟳ 执行中）
- 简化参数摘要（如“写入 src/components/Login.tsx”）
- 耗时（如“124ms”）
- 展开可查看完整参数与返回值
- 支持一键复制工具调用命令

**5.2.3 文件变更预览**

- 以类 Git Diff 的双栏对比视图展示 Agent 创建/修改的文件
- 绿色背景表示新增行，红色背景表示删除行
- 顶部工具栏显示文件路径、变更统计（+N / -M）
- 支持一键接受、拒绝或手动编辑变更

### 5.3 项目配置界面

**功能定位**：新建项目或调整现有项目配置的集中入口。

**5.3.1 基本信息区**

- 项目名称输入框
- 项目根目录选择器（带浏览按钮）
- 项目类型下拉选择：Web 应用 / CLI 工具 / API 服务 / 库
- 需求描述输入区：多行文本域，支持 Markdown，建议预置结构化提示模板

**5.3.2 技术栈配置**

采用卡片式多选或标签式选择：

- 前端框架：React / Vue / Next.js / 无
- 后端框架：Go / Express / Fastify / 无
- 数据库：SQLite / PostgreSQL / 无
- UI 组件库：shadcn/ui / Tailwind / 无

**5.3.3 Agent 团队配置**

- 以网格形式展示可选的 Agent 角色（产品经理、架构师、前端、后端、测试、运维、审查）
- 每个角色带开关（启用/禁用）
- 启用后展开模型选择下拉（Gemini 2.5 Pro / Gemini 2.5 Flash / Claude / 自定义）
- 支持为不同角色分配不同模型，以优化成本与效果

**5.3.4 MCP 服务器配置**

- 表格形式管理 MCP Server 连接
- 列：名称、传输类型（stdio/sse）、命令/URL、状态、操作
- 支持添加、编辑、删除、测试连接
- 连接成功后自动展示该 Server 提供的工具列表

**5.3.5 Skills 启用列表**

- 搜索/筛选可用 Skill
- 每个 Skill 卡片显示：名称、描述、触发条件、所需工具
- 开关控制是否在当前项目启用

**5.3.6 配置模板**

- 顶部提供「保存为模板」/「从模板加载」按钮
- 支持导入/导出 JSON/YAML 格式配置文件

### 5.4 技能管理界面

**功能定位**：管理 Skills 技能库，查看使用统计，编辑或创建新 Skill。

**5.4.1 技能库列表**

- 左侧为技能分类/搜索区
- 主区为技能卡片网格或表格
- 每个技能卡片显示：名称、版本、触发条件、使用次数、成功率、启用开关
- 排序选项：按使用次数、按更新时间、按名称

**5.4.2 技能详情/编辑器**

- 右侧或展开面板显示 Skill 完整内容
- 采用分栏预览模式：左侧为 Markdown 源码编辑，右侧为渲染后预览
- 顶部显示 YAML frontmatter 表单视图（可切换为源码编辑）
- 支持版本历史查看与回滚
- 提供“从执行轨迹生成”按钮，基于最近成功的执行记录自动生成 Skill 草稿

**5.4.3 使用统计视图**

- 展示技能使用趋势图（按日/周/月）
- 成功率趋势
- 平均节省 Token 数估算
- 关联执行记录列表

### 5.5 记忆查看器

**功能定位**：查看和管理三层记忆系统中的持久化内容。

**5.5.1 情节记忆检索**

- 搜索框支持全文检索（基于 FTS5）
- 结果以时间线卡片形式展示，每个卡片包含：任务类型、描述摘要、执行步骤概要、结果、时间
- 点击卡片展开完整记录
- 支持按 Agent 角色、任务类型、时间范围筛选
- 可手动删除或标记为“不相关”

**5.5.2 USER.md 编辑器**

- 分栏 Markdown 编辑器（源码+预览）
- 预置结构化模板：
  - 编码风格偏好
  - 常用技术栈
  - 命名规范
  - 备注/特殊要求
- 自动保存（防抖）

**5.5.3 项目 MEMORY.md 编辑器**

- 格式同 USER.md
- 内容示例：项目架构约定、已确认的技术决策、已知问题、重要上下文
- 支持多项目切换查看

### 5.6 设置界面

**功能定位**：全局应用设置与模型配置。

**5.6.1 通用设置**

- 主题选择：深色（默认）/ 浅色 / 跟随系统
- 语言选择：中文 / English
- 日志级别：DEBUG / INFO / WARN / ERROR
- 历史记录保留天数（默认 90 天）
- 自动更新开关

**5.6.2 模型配置**

- 表格形式管理模型提供商
- 每个提供商可配置：API Key（显示为“已设置/未设置”）、默认模型、高级参数（Temperature、MaxTokens 等）
- API Key 输入时提供「测试连接」按钮
- 支持添加自定义提供商（OpenAI 兼容 API）

**5.6.3 Agent 角色配置**

- 全局 Agent 角色模板管理
- 可为每个角色预设默认模型、默认提示词模板
- 支持导入/导出 Agent 配置


## 6. 组件设计规范

### 6.1 基础组件

**6.1.1 按钮**

| 变体     | 使用场景                       | 视觉规范                                       |
| -------- | ------------------------------ | ---------------------------------------------- |
| 主要按钮 | 页面核心操作（如“运行流水线”） | 主色背景 + 白色文字，圆角 6px，内边距 8px 16px |
| 次要按钮 | 辅助操作（如“取消”）           | 透明背景 + 主色边框 + 主色文字                 |
| 文本按钮 | 轻度操作（如“查看更多”）       | 无边框背景，悬停时显示浅主色背景               |
| 图标按钮 | 工具栏快捷操作                 | 24×24px 点击区域，悬停背景变化                 |
| 危险按钮 | 删除、重置等破坏性操作         | 红色背景/边框                                  |

**6.1.2 输入框**

- 圆角 6px，边框 1px（默认中性色，聚焦时变为主色）
- 内边距 8px 12px
- 支持前缀/后缀图标（如搜索图标、清除按钮）
- 错误状态：红色边框 + 下方错误提示文字

**6.1.3 下拉选择**

- 样式同输入框
- 展开菜单最大高度 300px，超出滚动
- 选项悬停背景变化
- 支持分组选项

**6.1.4 开关**

- 默认状态：灰色背景 + 白色圆点
- 开启状态：主色背景 + 白色圆点（右移）
- 过渡动画 150ms ease
- 支持禁用状态（半透明）

**6.1.5 复选框/单选框**

- 复选框：选中时主色背景 + 白色勾选图标
- 单选框：选中时主色外圈 + 内部主色圆点
- 标签支持点击选中

**6.1.6 标签**

- 用于状态标识（Agent 状态、项目类型等）
- 圆角 16px，内边距 4px 8px
- 支持多种语义色：默认/成功/警告/错误/信息
- 可关闭标签（带 × 图标）

**6.1.7 进度环**

- 用于 Agent 卡片中的任务进度
- 环形 SVG，主色填充
- 中心显示百分比数字（可选）
- 支持不确定进度（旋转动画）

**6.1.8 对话气泡**

- 用于思维链面板
- 圆角 16px，不同 Agent 使用不同边框/背景色
- 支持 Markdown 内容渲染
- 流式输出时带光标闪烁效果

### 6.2 复合组件

**6.2.1 Agent 状态卡片**

- 最小宽度 160px，高度 120px
- 包含：头像/图标、名称、角色、状态指示灯、任务简述、进度环/Token 消耗
- 运行中状态带边框脉冲光效
- 点击展开详情抽屉

**6.2.2 执行时间线**

- 基于 Canvas 或 SVG 绘制
- 支持缩放（时间粒度调整）和拖拽平移
- 悬停显示 Tooltip（详细信息）
- 点击时间块高亮对应 Agent

**6.2.3 文件树**

- 树形结构展示目录与文件
- 文件夹可展开/折叠
- 文件带图标（按扩展名区分）
- 支持右键菜单（打开、复制路径、在 Finder 中显示）
- 变更文件带状态标记（M 修改 / A 新增 / D 删除）

**6.2.4 Diff 视图**

- 双栏对比：左侧为原始内容（或空），右侧为修改后内容
- 行号显示
- 新增行绿色背景，删除行红色背景
- 差异块之间以虚线分隔
- 顶部工具栏：文件路径、变更统计、接受/拒绝按钮

**6.2.5 终端模拟器**

- 基于 Xterm.js 实现
- 深色背景（#1E1E1E），等宽字体
- 支持 ANSI 颜色转义序列
- 支持命令输入与执行
- 保留历史输出，支持清屏

**6.2.6 思维链气泡流**

- 对话式流式布局，新消息自动滚动到底部
- 每条气泡显示发送者（Agent 名称）和时间
- 支持 Markdown 及代码高亮
- 流式输出时显示“正在输入”动画
- 底部带输入框，支持用户追问

### 6.3 反馈组件

**6.3.1 通知**

- 右上角短暂出现（3–5 秒自动消失）
- 类型：成功（绿色）/ 信息（蓝色）/ 警告（黄色）/ 错误（红色）

**6.3.3 加载状态**

- 局部加载：小型旋转器，位于按钮或卡片内

### 7.2 字体排版

**7.2.1 字体族**

| 用途             | 字体栈                                                       |
| ---------------- | ------------------------------------------------------------ |
| 系统界面（中文） | `"PingFang SC", "Microsoft YaHei", "Source Han Sans CN", sans-serif` |
| 系统界面（英文） | `"Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |

**7.2.2 字体比例（Type Scale）**

基于 16px 基准，采用 1.25 比例（Major Third）：

| Token     | 大小 | 行高       | 使用场景            |
| --------- | ---- | ---------- | ------------------- |
| text-xs   | 12px | 1.5 (18px) | 辅助信息、标签      |
| text-sm   | 14px | 1.5 (21px) | 次要正文、Tooltip   |
| text-base | 16px | 1.5 (24px) | 正文、输入框、按钮  |
| text-lg   | 20px | 1.5 (30px) | 卡片标题、面板标题  |
| text-xl   | 24px | 1.4 (34px) | 页面标题            |
| text-2xl  | 32px | 1.3 (42px) | 主要标题            |
| text-3xl  | 40px | 1.2 (48px) | Hero 标题（欢迎页） |

代码编辑器内字体大小默认 14px，支持用户自定义调节。

### 7.3 图标规范

**7.3.1 图标系统**

使用 **Lucide React** 作为主图标库，覆盖绝大多数场景。

- 图标尺寸：16px（小）、20px（默认）、24px（大）、32px（特大）
- 线条粗细：1.5px–2px，与整体视觉保持一致
- 所有图标支持 fill 和 stroke 两种风格，界面主要使用 stroke 风格

**7.3.2 自定义图标**

以下场景需设计专属图标：

- 应用 Logo 与启动图标
- 各 Agent 角色头像图标（Product Manager、Architect、Frontend Dev 等）
- Skills 功能图标
- Memory 功能图标
- 状态指示图标（成功/失败/警告/运行中等动画图标）

**7.3.3 图标使用原则**

- 尽量搭配文字标签，避免仅依靠图标传达含义
- 悬停时显示 Tooltip 说明功能
- 危险操作图标使用红色强调

### 7.4 动效设计

**7.4.1 动效原则**

- 快速响应：过渡时长通常在 150ms–250ms 之间
- 自然缓动：使用 `ease-out`（入场）和 `ease-in`（退场）
- 不干扰：动效应辅助传达信息，而非分散注意力

**7.4.2 常用动效**

| 场景           | 时长       | 缓动             | 说明               |
| -------------- | ---------- | ---------------- | ------------------ |
| 悬停状态变化   | 150ms      | ease-out         | 背景色、边框色变化 |
| 面板展开/折叠  | 250ms      | ease-in-out      | 宽度变化           |
| 模态框入场     | 200ms      | ease-out         | 缩放+淡入          |
| 通知入场/退场  | 300ms      | ease-out/ease-in | 滑入滑出           |
| Agent 运行脉冲 | 1.5s       | ease-in-out      | 无限循环边框光效   |
| 流式文字       | 逐字       | —                | 模拟打字效果       |
| 进度环填充     | 与进度同步 | linear           | —                  |

**7.4.3 动画令牌（Motion Tokens）**

```css
--duration-instant: 100ms;
--duration-fast: 150ms;
--duration-base: 200ms;
--duration-slow: 300ms;

--ease-standard: cubic-bezier(0.2, 0, 0, 1);
--ease-emphasized: cubic-bezier(0.05, 0.7, 0.1, 1);
```

### 7.5 阴影与层级

**7.5.1 阴影定义（深色模式）**

| 层级 | 阴影值                                                       | 使用场景          |
| ---- | ------------------------------------------------------------ | ----------------- |
| sm   | `0 1px 2px 0 rgba(0,0,0,0.4)`                                | 卡片、按钮        |
| md   | `0 4px 6px -2px rgba(0,0,0,0.4), 0 2px 4px -1px rgba(0,0,0,0.3)` | 下拉菜单、Tooltip |
| lg   | `0 10px 15px -3px rgba(0,0,0,0.5), 0 4px 6px -2px rgba(0,0,0,0.3)` | 模态框            |
| xl   | `0 20px 25px -5px rgba(0,0,0,0.5), 0 8px 10px -3px rgba(0,0,0,0.4)` | 抽屉面板          |

**7.5.2 液态玻璃效果（Liquid Glass）**

用于悬浮组件（如命令面板、快速操作菜单），营造精致现代感：

- 背景色：半透明白/黑（`rgba(255,255,255,0.08)` 或 `rgba(0,0,0,0.6)`）
- 背景模糊：`backdrop-filter: blur(12px)`
- 边框：半透明白色描边（`rgba(255,255,255,0.1)`）
- 阴影：md 级别


## 8. 可访问性设计

### 8.1 颜色对比度

确保文字与背景的对比度满足 WCAG 2.2 AA 级标准：

| 文字类型          | 最小对比度 | 验证方式                 |
| ----------------- | ---------- | ------------------------ |
| 正常文字（<18px） | ≥ 4.5:1    | 主要正文、标签、按钮文字 |
| 大文字（≥18px）   | ≥ 3:1      | 页面标题、卡片标题       |
| 图标与图形        | ≥ 3:1      | UI 图标、状态指示器      |

### 8.2 键盘导航

- 支持 Tab 键在可聚焦元素间顺序切换
- 支持 Enter / Space 激活按钮和开关
- 支持 ESC 关闭弹窗和菜单
- 支持方向键在列表、菜单、选项卡间导航
- 提供“跳过导航”链接（用于屏幕阅读器）
- 焦点指示器清晰可见（主色轮廓，2px）

### 8.3 屏幕阅读器支持

- 所有非文本内容提供替代文本（`aria-label` 或 `alt`）
- 动态内容更新使用 `aria-live` 区域（如日志流、思维链流）
- 表单控件关联正确的 `<label>` 和 `aria-describedby`
- 自定义组件（如开关、进度环）提供正确的 ARIA 角色和状态

### 8.4 触摸与指针

- 所有可点击元素最小点击区域 44×44px
- 支持触摸板手势（双指滚动、捏合缩放等）
- 拖拽操作支持触摸拖动


## 9. 设计令牌（Design Tokens）

### 9.1 令牌结构

采用 W3C Design Tokens 规范的三层结构：Reference → System → Component。

```
tokens/
├── reference/
│   ├── color-primitive.json    # 基础色值
│   ├── spacing-primitive.json  # 基础间距
│   └── typography-primitive.json
├── system/
│   ├── color-semantic.json     # 语义色
│   ├── motion.json             # 动效时长与缓动
│   └── elevation.json          # 阴影与层级
└── component/
    ├── button.json
    ├── card.json
    └── ...
```

### 9.2 令牌示例

**颜色令牌（color-semantic.json）：**
```json
{
  "color": {
    "background": {
      "page": {"$type": "color", "$value": "{color.primitive.gray.0}"},
      "card": {"$type": "color", "$value": "{color.primitive.gray.1}"},
      "hover": {"$type": "color", "$value": "{color.primitive.gray.2}"}
    },
    "text": {
      "primary": {"$type": "color", "$value": "{color.primitive.gray.6}"},
      "secondary": {"$type": "color", "$value": "{color.primitive.gray.4}"}
    }
  }
}
```

**间距令牌（spacing-system.json）：**
```json
{
  "spacing": {
    "xs": {"$type": "dimension", "$value": "4px"},
    "sm": {"$type": "dimension", "$value": "8px"},
    "md": {"$type": "dimension", "$value": "16px"},
    "lg": {"$type": "dimension", "$value": "24px"},
    "xl": {"$type": "dimension", "$value": "32px"}
  }
}
```


## 10. 前端技术实现

### 10.1 技术栈

| 层级       | 技术                       | 用途                  |
| ---------- | -------------------------- | --------------------- |
| UI 框架    | React 18 + TypeScript      | 组件开发              |
| 样式方案   | Tailwind CSS + CSS Modules | 样式管理              |
| 组件库基础 | shadcn/ui (Radix UI 原语)  | 基础交互组件          |
| 图标库     | Lucide React               | 图标系统              |
| 终端模拟   | Xterm.js                   | 终端面板              |
| 代码高亮   | Shiki                      | Diff 视图与日志代码块 |
| 图表绘制   | Recharts / Visx            | 时间线与统计图        |
| 动画       | Framer Motion              | 组件动效              |
| 状态管理   | Zustand                    | 前端全局状态          |
| 拖拽交互   | dnd-kit                    | 面板拖拽、列表排序    |

### 10.2 主题切换实现

- 使用 Tailwind CSS 的 `dark:` 变体配合 CSS 变量实现
- 主题配置集中管理，通过 `ThemeProvider` 注入
- 支持“跟随系统”模式（使用 `prefers-color-scheme` 媒体查询）
- 主题切换时保存用户偏好至 `localStorage`

### 10.3 与 Go 后端的实时通信

- 通过 Wails 的 `EventsOn` 订阅后端事件流
- Agent 状态、思维链、日志等以事件形式实时推送
- 前端维护乐观更新状态，保证 UI 流畅响应
- WebSocket 断线时自动重连，并显示连接状态提示
