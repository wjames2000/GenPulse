# 2.1.3 Agent间调用机制 - 实现说明

## 概述
已成功实现基于Genkit Tool的Agent编排能力，通过将Agent封装为Tool，实现了Agent间的层级调用机制。

## 实现文件

### 1. `internal/agents/agent_tool.go`
**核心功能**: 将Agent封装为Tool的适配器

**主要特性**:
- 实现`AgentTool`结构体，包装Agent并提供Tool接口
- 自动生成工具定义，包含Agent的元数据
- 支持参数验证和执行统计
- 与Agent状态同步（启用/禁用）

**关键方法**:
- `NewAgentTool()`: 创建AgentTool实例
- `Execute()`: 执行Agent任务，返回Tool格式的结果
- `ValidateParameters()`: 验证输入参数
- 状态管理方法：`IsEnabled()`, `SetEnabled()`
- 统计方法：`GetExecutionCount()`, `GetAverageDuration()`

### 2. `internal/agents/manager.go` (修改)
**新增功能**: Agent管理器自动注册Agent为Tool

**修改内容**:
- `registerAgentsAsTools()`: 初始化时将已注册的Agent自动注册为Tool
- `registerAgentAsTool()`: 注册单个Agent为Tool
- `RegisterAgent()`: 注册Agent时自动创建对应的Tool
- `UnregisterAgent()`: 注销Agent时同时注销对应的Tool

### 3. `internal/genkit/tools/tool_registry.go` (修改)
**新增辅助方法**:
- `IncrementExecutionCount()`: 供子类更新执行统计
- `Mutex()`: 供子类访问互斥锁

## 架构设计

### Agent → Tool 转换流程
```
Agent (实现Agent接口)
    ↓
AgentTool (实现Tool接口，包装Agent)
    ↓
ToolRegistry (注册和管理所有工具)
    ↓
其他Agent/Flow (通过Tool ID调用)
```

### 数据流示例
```
Agent A → 调用 ToolRegistry → AgentTool B → 执行 Agent B → 返回结果
```

## 核心特性

### 1. 自动注册机制
- Agent管理器初始化时自动将所有Agent注册为Tool
- 新注册的Agent会自动创建对应的Tool
- 注销Agent时会同步注销对应的Tool

### 2. 参数传递
- 支持任务描述和参数的传递
- 支持上下文信息的传递
- 参数验证确保输入格式正确

### 3. 状态同步
- Agent的启用/禁用状态与Tool同步
- 禁用Agent的Tool无法执行
- 状态变更实时生效

### 4. 执行统计
- 记录执行次数、平均耗时、最后执行时间
- 统计信息可用于监控和优化

### 5. 错误处理
- 统一的错误格式
- 详细的错误信息和上下文
- 执行失败时返回清晰的错误信息

## 使用示例

### 基本使用
```go
// 创建Agent
agent, _ := agents.NewBaseAgent(config, modelAdapter, toolRegistry, flowEngine)

// 创建AgentTool
agentTool, _ := agents.NewAgentTool(agent)

// 注册到ToolRegistry
toolRegistry.RegisterTool(agentTool)

// 通过Tool调用Agent
execution := tools.ToolExecution{
    ToolID: "agent_agent_id",
    Parameters: map[string]interface{}{
        "task": "执行任务",
        "parameters": map[string]interface{}{
            "param1": "value1",
        },
    },
}

result, _ := tool.Execute(ctx, execution)
```

### 通过AgentManager自动注册
```go
// 创建Agent管理器
agentManager := agents.NewAgentManager(modelAdapter, toolRegistry, flowEngine)

// 注册Agent
agentManager.RegisterAgent(agent)

// 初始化（自动注册为Tool）
agentManager.Initialize()

// 现在可以通过ToolRegistry调用Agent
tool, _ := toolRegistry.GetTool("agent_agent_id")
```

## 扩展性设计

### 支持多种调用模式
1. **直接调用**: 通过ToolRegistry直接调用AgentTool
2. **Flow编排**: 在Flow中调用AgentTool
3. **链式调用**: Agent A → AgentTool B → Agent C

### 未来扩展
1. **批量调用**: 支持同时调用多个AgentTool
2. **条件执行**: 基于执行结果的条件分支
3. **并行执行**: 多个AgentTool并行执行
4. **结果聚合**: 多个Agent执行结果的聚合

## 性能考虑

### 轻量级包装
- AgentTool是轻量级包装器，开销极小
- 执行统计使用原子操作，避免锁竞争
- 参数验证快速失败，减少不必要的执行

### 内存管理
- AgentTool不复制Agent，只持有引用
- 结果序列化按需进行
- 统计信息使用基本类型，内存占用小

## 安全性

### 参数验证
- 验证必需参数
- 验证参数类型
- 防止无效输入

### 访问控制
- 通过启用/禁用状态控制访问
- 支持基于角色的访问控制（通过Agent配置）
