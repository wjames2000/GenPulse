# GenPulse Go项目结构规划

## 项目根目录结构
```
GenPulse/
├── app.go              # Wails主应用结构
├── main.go            # 程序入口
├── go.mod             # Go模块定义
├── go.sum             # 依赖锁文件
├── wails.json         # Wails配置文件
├── frontend/          # 前端代码(React + TypeScript + Vite)
├── build/             # 构建配置
├── internal/          # 内部包(不对外暴露)

├── prompts/           # 提示词模板
├── skills/            # 技能定义文件
├── scripts/           # 构建和部署脚本
└── docs/              # 项目文档
```

## 模块依赖关系

```
frontend (React) 
    ↓ (Wails绑定)
app.go (Wails主应用)
    ↓
internal/services/ (服务层)
    ↓
internal/genkit/ (业务逻辑层)
    ├── internal/genkit/agents/    ← internal/genkit/tools/
    ├── internal/genkit/flows/     ← internal/genkit/agents/
    ├── internal/genkit/memory/    ← internal/database/
    ├── internal/genkit/skills/    ← internal/genkit/memory/
    └── internal/genkit/monitoring/ ← internal/events/
        ↓
internal/database/ (数据层)
internal/events/ (事件层)
```

## 包导入规则

1. **内部包导入**: 所有internal包只能被本项目内部导入
2. **层级依赖**: 上层可以导入下层，下层不能导入上层
3. **循环依赖**: 严格禁止循环依赖
4. **接口隔离**: 通过接口定义依赖，减少具体实现耦合

## 代码组织原则

1. **单一职责**: 每个文件/包只负责一个明确的功能
2. **接口先行**: 先定义接口，再实现具体类型
3. **依赖注入**: 通过构造函数注入依赖
4. **错误处理**: 统一错误处理机制
5. **日志记录**: 结构化日志记录
6. **配置管理**: 外部化配置管理
