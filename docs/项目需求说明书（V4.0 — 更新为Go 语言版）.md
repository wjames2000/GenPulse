# 项目需求说明书（V4.0 — 更新为Go 语言版）

## AI 软件项目开发流水线（桌面版）


### 一、版本更新说明

| 版本     | 日期           | 主要变更                                                 |
| -------- | -------------- | -------------------------------------------------------- |
| V1.0     | 2026-04-18     | 初始版本，基于 Genkit Node.js + Electron 架构            |
| V2.0     | 2026-04-18     | 整合 Gemini 模型评测数据与 Genkit 多 Agent 最佳实践      |
| V3.0     | 2026-04-18     | 整合 Hermes Agent 自进化理念（Skills 闭环 + 三层记忆）   |
| **V4.0** | **2026-04-18** | **技术栈迁移至 Go 语言：Genkit Go SDK + Wails 桌面框架** |

**核心变更摘要**：

- **AI 框架**：从 Genkit Node.js 迁移至 **Genkit Go SDK**
- **桌面框架**：从 Electron 迁移至 **Wails**（Go 后端 + 前端 Web 技术）
- **前端界面**：继续使用 React + TypeScript，通过 Wails 与 Go 后端通信
- **Git 操作**：从 simple-git 迁移至 **go-git**（纯 Go 实现）
- **MCP 集成**：使用 **Genkit MCP Go 插件** + **官方 go-sdk**
- **监控追踪**：使用 Genkit Go 内置的 **OpenTelemetry** 支持
- **进程管理**：使用 Go 标准库 `os/exec`


### 二、项目概述

> **注**：本节内容（项目背景、设计哲学、项目目标、核心场景）与 V3.0 保持一致，此处仅保留要点摘要。

#### 2.1 设计哲学（更新）

本项目延续 V3.0 的双层架构设计哲学，但在技术实现上全面迁移至 Go 语言生态：

| 架构范式              | 核心特点                                                   | Go 版本实现方式                                 |
| --------------------- | ---------------------------------------------------------- | ----------------------------------------------- |
| **Orchestrator 型**   | 中心网关负责路由和调度；生态成熟、团队协作友好             | Genkit Go SDK 的 Flow + Agent 编排能力          |
| **Learning Agent 型** | Agent 逐步进化的“长期协作者”；关注记忆、技能抽取与自动改进 | Go 实现的自进化子系统（Skills 闭环 + 三层记忆） |

#### 2.2 技术栈变更对比

| 层级            | V3.0（Node.js/TypeScript） | V4.0（Go）                         | 变更说明                                                     |
| --------------- | -------------------------- | ---------------------------------- | ------------------------------------------------------------ |
| **AI 编排框架** | Genkit Node.js             | **Genkit Go SDK**                  | Genkit 提供 TypeScript 与 Go 并行实现，Go 版本同样支持 Flow、Tool、Prompt 等核心能力 |
| **桌面框架**    | Electron                   | **Wails**                          | Wails 采用 Go 后端 + Web 前端架构，打包体积仅 10–20MB，启动速度 <80ms，远优于 Electron 的上百 MB 体积 |
| **前端界面**    | React + TypeScript         | React + TypeScript（不变）         | 前端技术栈保持不变，通过 Wails 绑定与 Go 后端通信            |
| **状态管理**    | Zustand / Redux            | Zustand / Redux（不变）            | 纯前端部分不变                                               |
| **文件操作**    | Node.js fs API             | Go `os` + `path/filepath`          | Go 标准库提供完整的文件系统操作                              |
| **Git 操作**    | simple-git                 | **go-git**                         | go-git 是纯 Go 实现的 Git 库，提供 porcelain（高层）和 plumbing（低层）API，完全兼容 Git 命令行行为 |
| **进程管理**    | child_process              | `os/exec`                          | Go 标准库执行外部命令                                        |
| **数据持久化**  | SQLite / LevelDB           | **SQLite**（使用 go-sqlite3 驱动） | Go 生态有成熟的 SQLite 驱动                                  |
| **MCP 集成**    | @genkit-ai/mcp（npm）      | **Genkit MCP Go 插件**             | Genkit 为 Go 提供 MCP 插件，支持 MCP 工具消费和暴露          |
| **监控追踪**    | OpenTelemetry（Node.js）   | **OpenTelemetry（Go）**            | Genkit Go 内置 OpenTelemetry 插桩，支持 Trace 导出和 Dev UI  |

#### 2.3 为什么选择 Go 语言

**技术优势**：

1. **性能卓越**：Go 是编译型语言，执行效率远高于 JavaScript，特别适合计算密集型的 AI 应用后台。
2. **并发模型优秀**：Go 的 goroutine 和 channel 天然适合多 Agent 并行执行的场景。
3. **部署简单**：Go 编译为单一可执行文件，无需运行时依赖，分发和部署极为便捷。
4. **类型安全**：编译期类型检查，减少运行时错误，提升代码可靠性。
5. **跨平台编译**：Go 原生支持交叉编译，可同时生成 Windows/macOS/Linux 可执行文件。

**Genkit Go 的成熟度**：

- Genkit 提供 JavaScript/TypeScript、Go 和 Python 三种语言的并行实现，共享相同的架构模式
- Go 版本支持 Flow、Prompt、Tool 等核心编排原语，提供类型安全的 AI 工作流定义能力
- Go 版本内置 OpenTelemetry 追踪支持，可导出到监控系统
- Google 还推出了专为 Go 设计的 **Agent Development Kit (ADK)**，支持构建模块化多 Agent 系统，与 Genkit 生态互补

**Wails 桌面框架的优势**：

- **体积轻量**：打包体积仅 10–20MB，而 Electron 通常上百 MB
- **启动快速**：启动速度不足 80ms
- **前后端分离**：用 React/Vue 写界面，用 Go 写业务逻辑，各取所长
- **原生通信**：提供优雅的 Go ↔ JS 绑定机制，前端可直接调用 Go 函数
- **跨平台支持**：支持 Windows/macOS/Linux 原生应用生成

### 四、核心功能模块需求（Go 实现版）

> **注**：以下仅列出因技术栈变更而需要调整或新增的内容，与 V3.0 一致的功能模块（如 Skills 闭环阶段定义、三层记忆模型、Agent 角色定义等）保持不变。

#### 4.1 文件系统操作模块（Go 实现）

基于 Go 标准库 `os` 和 `path/filepath` 实现，通过 Genkit Tool 机制暴露给 Agent：

| 工具名      | 功能             | Go 实现方式                      |
| ----------- | ---------------- | -------------------------------- |
| `fs_cwd`    | 获取当前工作目录 | `os.Getwd()`                     |
| `fs_list`   | 列出目录内容     | `os.ReadDir()`                   |
| `fs_read`   | 读取文件内容     | `os.ReadFile()`                  |
| `fs_write`  | 写入文件         | `os.WriteFile()` + 权限控制      |
| `fs_delete` | 删除文件/目录    | `os.Remove()` / `os.RemoveAll()` |
| `fs_mkdir`  | 创建目录         | `os.MkdirAll()`                  |
| `fs_copy`   | 复制文件/目录    | 自定义实现（使用 `io.Copy`）     |
| `fs_move`   | 移动文件/目录    | `os.Rename()`                    |

**安全增强**：所有路径操作前使用 `filepath.Abs()` 规范化，并通过 `strings.HasPrefix()` 校验是否在项目根目录内，防止路径穿越攻击。

#### 4.2 Git 操作模块（go-git）

使用纯 Go 实现的 `go-git` 库：

| 工具名       | 功能             | go-git API                                    |
| ------------ | ---------------- | --------------------------------------------- |
| `git_init`   | 初始化 Git 仓库  | `git.PlainInit()`                             |
| `git_clone`  | 克隆远程仓库     | `git.PlainClone()`                            |
| `git_status` | 查看工作区状态   | `worktree.Status()`                           |
| `git_add`    | 添加文件到暂存区 | `worktree.Add()`                              |
| `git_commit` | 提交更改         | `worktree.Commit()`                           |
| `git_branch` | 创建/切换分支    | `repo.CreateBranch()` / `worktree.Checkout()` |
| `git_push`   | 推送到远程仓库   | `repo.Push()`                                 |
| `git_pull`   | 拉取远程更新     | `worktree.Pull()`                             |
| `git_log`    | 查看提交历史     | `repo.Log()`                                  |

#### 4.3 命令行执行模块（os/exec）

使用 Go 标准库 `os/exec`：

| 工具名         | 功能                      | Go 实现方式                               |
| -------------- | ------------------------- | ----------------------------------------- |
| `shell_exec`   | 执行 shell 命令并返回输出 | `exec.Command().CombinedOutput()`         |
| `shell_spawn`  | 启动长期运行的进程        | `exec.Command().Start()` + goroutine 管理 |
| `npm_install`  | 安装 npm 依赖             | 封装 `shell_exec` 调用 `npm install`      |
| `npm_run`      | 运行 npm scripts          | 封装 `shell_exec` 调用 `npm run`          |
| `process_kill` | 终止指定进程              | `cmd.Process.Kill()`                      |

**安全控制**：维护命令白名单，限制可执行的二进制路径；危险命令需用户二次确认。

#### 4.4 MCP 集成模块（Go 实现）

使用 Genkit Go MCP 插件集成 Model Context Protocol。该插件支持在 Genkit 应用中消费 MCP 工具、资源和提示词，同时也可将 Genkit 组件暴露为 MCP 服务器供其他客户端使用。

| 功能点         | Go 实现方式                                                  |
| -------------- | ------------------------------------------------------------ |
| MCP Client     | 使用 Genkit MCP 插件的 `createMcpHost` 函数连接外部 MCP Server |
| MCP Server     | 使用 Genkit 插件机制暴露本地工具为 MCP Server                |
| 多 Server 管理 | 通过 MCPHost 管理多个连接，支持命名空间隔离                  |
| 工具发现       | Genkit MCP 插件自动从 MCP Server 发现工具并创建命名空间化的 Genkit Action |

**Go 版 MCP 集成优势**：

- Genkit 官方提供 Go 版本的 MCP 插件，无需第三方库
- 与 Genkit Flow 和 Tool 系统深度集成，工具调用无缝
- 支持 `stdio`、`sse` 等多种传输协议

#### 4.5 监控与追踪模块（OpenTelemetry Go）

Genkit Go 使用 OpenTelemetry 进行了全面插桩，并提供了用于导出遥测数据的钩子：

| 功能点      | Go 实现方式                                                  |
| ----------- | ------------------------------------------------------------ |
| Trace 采集  | Genkit 自动为 Action 和 Flow 创建 OpenTelemetry Span，捕获执行路径、输入输出和耗时 |
| 自定义 Span | 使用 `core.Run()` 或 `tracing.RunInNewSpan()` 创建追踪子步骤 |
| Trace 导出  | 通过 `tracing.WriteTelemetryImmediate()` 或 `WriteTelemetryBatch()` 配置导出 |
| Dev UI 集成 | 设置 `GENKIT_ENV=dev` 后 Trace 自动发送到 Genkit Developer UI |
| 遥测插件    | 可通过编写 Genkit 遥测插件导出到 Google Cloud Monitoring 或其他监控系统 |

**Go 版本监控优势**：

- 编译时类型安全，减少运行时错误
- 与 Go 标准 `context.Context` 无缝集成
- 支持批量导出，网络效率更高

### 七、非功能性需求（Go 版更新）

| 需求项         | V3.0 要求                  | V4.0 更新                                            |
| -------------- | -------------------------- | ---------------------------------------------------- |
| **启动时间**   | 应用冷启动 < 5 秒          | < 2 秒（Go 编译二进制启动极快）                      |
| **内存占用**   | 空闲 < 500MB，运行中 < 2GB | 空闲 < 100MB，运行中 < 1GB（Wails 比 Electron 轻量） |
| **打包体积**   | 未明确                     | 安装包 < 30MB（Wails 产物典型体积）                  |
| **并发 Agent** | 支持 5+ Agent              | 支持 10+ Agent（Go goroutine 并发模型优势）          |
| **跨平台支持** | Windows/macOS/Linux        | 相同，Go 交叉编译更简单                              |


### 八、风险与应对（Go 版新增）

| 风险                             | 影响     | 概率 | 应对措施                                           |
| -------------------------------- | -------- | ---- | -------------------------------------------------- |
| Go 生态部分插件不如 Node.js 丰富 | 功能受限 | 低   | Genkit Go 官方插件覆盖核心需求，MCP 可扩展任意能力 |
| Wails 前端绑定调试难度           | 开发效率 | 中   | Wails 提供 dev 模式热重载，官方文档完善            |
| go-git 与 Git 命令行行为差异     | 操作异常 | 低   | go-git 保证 porcelain 操作与 Git 完全兼容          |
| 团队 Go 技能曲线                 | 开发进度 | 中   | Go 语言学习曲线平缓，1-2 周可上手                  |
| OpenTelemetry Go SDK 集成复杂度  | 监控延迟 | 低   | Genkit 已内置 OpenTelemetry 插桩，开箱即用         |
