# 项目需求说明书（V3.0 — 整合 Hermes Agent 理念版）

## AI 软件项目开发流水线（桌面版）


### 一、项目概述

#### 1.1 项目背景

当前AI辅助软件开发工具虽然层出不穷，但实现“从0到1全自动写项目”仍面临以下核心挑战：

- **流水线缺失**：现有工具缺乏系统级的全流程自动化编排能力。
- **协作孤岛**：不同AI角色之间的协作需要大量人工协调。
- **能力固化**：Agent的技能依赖开发者手动编写，面对动态变化的任务时技能库迅速失效。
- **记忆断层**：传统Agent“无状态”——每次对话从零开始，做完就忘，无法积累项目经验。
- **操作受限**：对本地文件系统、Git和命令行的深度操作能力不足。
- **过程黑盒**：缺乏对AI思考链、工具调用和资源消耗的实时监控。

#### 1.2 设计哲学：融合两种架构范式

本项目的设计哲学受到两种主流Agent架构范式的启发，取其精华、融合创新：

| 架构范式                                       | 核心特点                                                     | 本项目的借鉴                                     |
| ---------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------ |
| **Orchestrator型**（如OpenClaw/Genkit多Agent） | 中心网关负责路由和调度；生态成熟、团队协作友好               | 采用Genkit的`Agent-as-Tool`层级化多Agent编排能力 |
| **Learning Agent型**（如Hermes Agent）         | Agent本身是逐步进化的“长期协作者”；关注记忆、技能抽取与自动改进 | 引入自进化技能系统和持久化记忆架构               |

许多前沿团队的实践路径是：用Orchestrator型框架做编排层（统一入口、权限、监控），用Learning Agent型作为专精型Agent，解决那些高度重复、需要长期记忆和技能演化的任务。本项目正是沿着这一路径，将Genkit的强大编排能力与Hermes式的自进化能力深度融合。

#### 1.3 项目目标

1. **全自动项目生成**：用户输入需求描述，系统自动完成从项目初始化到可运行代码的全流程生成。
2. **多Agent协作**：支持配置多个角色Agent，各司其职、协同工作。
3. **技能自进化**：借鉴Hermes Agent的设计理念，让Agent在开发过程中**自动沉淀经验为Skill**，实现“越用越聪明”的持续进化能力。
4. **持久化记忆**：构建跨会话的记忆系统，让Agent记住项目上下文、编码习惯和技术决策，告别“每次从零开始”。
5. **能力无限扩展**：通过MCP协议和Skills机制无限扩展Agent的工具调用能力。
6. **全方位监控**：可视化监控每个Agent的执行状态、思维链和资源消耗。
7. **本地环境操作**：Agent可直接操作本地文件系统和Git仓库。

#### 1.4 核心场景

**“全自动从0到1写项目”**：用户输入需求描述 → 系统自动完成需求分析 → 技术方案设计 → 项目脚手架生成 → 代码实现 → 测试编写 → 项目运行验证，全程无需人工干预。更关键的是，项目执行过程中**自动沉淀经验为Skill**，下次类似任务直接复用，速度提升可达40%。

#### 2.2 架构核心理念：编排层 + 进化层

本项目采用**双层架构**设计：

- **编排层（Genkit）**：负责多Agent调度、流水线编排、工具调用分发和监控。这是系统的“骨架”，确保多个Agent能够高效协作完成复杂任务。
- **进化层（自进化子系统）**：借鉴Hermes Agent的设计理念，实现技能闭环和持久化记忆。这是系统的“大脑”，让Agent在每次任务中积累经验，实现“越用越强”。

这种双层架构的优势在于：
- **Genkit提供生产级的Agent编排能力**（类型安全、Flow定义、MCP集成、OpenTelemetry监控）。
- **自进化子系统提供持续的技能成长**，让每个专业Agent都能沉淀领域知识，逐步变成该领域的“专家”。

#### 2.3 技术栈选型

| 层级           | 技术选型                    | 核心作用                                |
| -------------- | --------------------------- | --------------------------------------- |
| **桌面框架**   | Electron                    | 跨平台桌面应用（Windows/macOS/Linux）   |
| **AI编排核心** | **Google Genkit** (Node.js) | 多Agent调度、Flow编排、MCP集成          |
| **前端界面**   | React + TypeScript          | 动态高性能用户界面                      |
| **技能存储**   | 文件系统 + SQLite           | 存储Skill文档（遵循agentskills.io标准） |
| **记忆存储**   | SQLite + 全文检索（FTS5）   | 三层记忆架构的持久化存储                |
| **文件操作**   | Node.js fs API + genkitx-fs | 安全可控的文件系统读写                  |
| **Git操作**    | simple-git                  | Node.js Git操作库                       |

| **通信协议**   | IPC + WebSocket             | 主进程与渲染进程通信、实时状态推送      |

#### 2.4 为什么选择Genkit + 自进化子系统

**Genkit的优势**：
- 类型安全的Flow定义，确保输入输出可预测
- 层级化多Agent架构（Agent-as-Tool），实现清晰的分层调度
- 原生MCP集成，无限扩展工具能力
- OpenTelemetry集成，便于监控和追踪

**自进化子系统的引入原因**：
- 突破传统Agent依赖人工编写Skill的瓶颈
- 让Agent在项目开发中自动积累领域经验，形成可复用的“方法资产库”
- 通过三层记忆架构实现跨会话的上下文保持，告别“每次从零开始”


### 三、核心功能模块需求

#### 3.1 技能自进化系统（Skills闭环）

**设计理念**：借鉴Hermes Agent的核心理念——让Agent“自己长本事”，而非依赖开发者手动编写Skill。

**3.1.1 Skills闭环的七个阶段**

系统将实现从经验提取到知识复用的完整生命周期：

```
┌─────────────────────────────────────────────────────────────────┐
│                    Skills 闭环系统                               │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐         │
│  │ 1.经验  │ → │ 2.触发  │ → │ 3.提取  │ → │ 4.结构化 │         │
│  │   积累  │   │   判断  │   │   流程  │   │   生成  │         │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘         │
│       ↑                                             ↓           │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐         │
│  │ 7.持续  │ ← │ 6.验证  │ ← │ 5.存储  │ ← │   版本  │         │
│  │   优化  │   │   执行  │   │   管理  │   │   管理  │         │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘         │
└─────────────────────────────────────────────────────────────────┘
```

**各阶段详细说明**：

| 阶段 | 名称       | 功能说明                                      | 触发条件                                               |
| ---- | ---------- | --------------------------------------------- | ------------------------------------------------------ |
| 1    | 经验积累   | Agent在执行任务中调用工具、遭遇错误、调整策略 | 自动进行                                               |
| 2    | 触发判断   | Agent自主判断是否需要创建Skill                | 复杂任务（5+工具调用）、修复棘手错误、发现非平凡工作流 |
| 3    | 流程提取   | 从成功经验中提取关键步骤、常见陷阱、验证方法  | 触发后自动执行                                         |
| 4    | 结构化生成 | 生成符合agentskills.io标准的SKILL.md文档      | 调用skill_manage工具                                   |
| 5    | 存储管理   | 保存到本地技能库，支持版本管理                | 写入~/.hermes/skills/                                  |
| 6    | 验证执行   | 下次类似任务时加载Skill，跳过摸索阶段         | 任务匹配时自动触发                                     |
| 7    | 持续优化   | 发现更优方法或过时内容时自动更新Skill         | 使用中发现改进空间时                                   |

**3.1.2 Skill文档标准**

遵循开源的**agentskills.io**标准，确保技能可跨平台移植。每个Skill文档包含：

- **YAML frontmatter**：名称、版本、触发条件、所需工具、适用场景等元数据
- **Markdown正文**：完整的操作步骤、已知边界条件、验证方法、常见陷阱

**示例结构**：
```yaml
---
name: react-component-generator
version: 2
trigger: "生成React组件"
tools_required: [fs_write, fs_read, npm_run]
---
# React 组件生成工作流
## 步骤
1. 分析组件需求，确定Props接口
2. 生成组件TSX代码，包含类型定义
3. 生成对应的样式文件
4. 生成单元测试骨架
5. 更新导出索引文件
## 注意事项
- 遵循Airbnb React/JSX风格指南
- 组件命名使用PascalCase
- Props接口以ComponentNameProps命名
```

**3.1.3 技能加载优化：渐进式披露**

为节约Token消耗，系统采用**渐进式披露**策略：

- **Level 0**：仅加载Skill列表（名称+触发条件），用于任务匹配
- **Level 1**：匹配成功后加载完整Skill详情，注入执行上下文

这种设计既保证了技能匹配的覆盖面，又控制了每次调用的Token成本。

**3.1.4 技能使用效果预期**

基于社区数据，使用自生成Skill后研究任务提速约40%。在项目开发场景中，预期效果：

| 场景             | 首次执行       | 复用Skill后 | 提升幅度 |
| ---------------- | -------------- | ----------- | -------- |
| 生成React组件    | 5-8轮工具调用  | 1-2轮       | ~60%     |
| 搭建API端点      | 6-10轮工具调用 | 2-3轮       | ~55%     |
| 配置数据库Schema | 4-6轮工具调用  | 1轮         | ~70%     |
| 编写单元测试     | 3-5轮工具调用  | 1轮         | ~65%     |

#### 3.2 三层记忆架构

**设计理念**：借鉴Hermes Agent的记忆设计——不是简单的“记住对话”，而是一个分层、持久化的知识系统。让Agent不仅能记住“说了什么”，更能理解“为什么这么做”和“怎么做更好”。

**3.2.1 三层记忆模型**

| 记忆层 | 名称     | 功能                             | 存储内容示例                                 | 存储方式                |
| ------ | -------- | -------------------------------- | -------------------------------------------- | ----------------------- |
| **L1** | 工作记忆 | 当前会话上下文、即时工具调用结果 | 正在编写的文件内容、刚执行的命令输出         | 内存（会话级）          |
| **L2** | 情节记忆 | 跨会话保留的任务执行记录         | “上周成功部署了一个Next.js项目到Vercel”      | SQLite + 全文检索       |
| **L3** | 语义记忆 | 持久化的知识、用户偏好、项目画像 | 用户的编码风格偏好、常用技术栈、项目架构约定 | MEMORY.md / USER.md文件 |

**3.2.2 记忆存储实现**

- **MEMORY.md**：记录项目相关的事实、技术决策、已知问题
- **USER.md**：构建用户画像，记录编码习惯、偏好框架、命名规范等
- **SQLite + FTS5**：实现情节记忆的持久化存储和全文检索，支持按任务类型、时间、关键词快速检索历史经验

**3.2.3 记忆应用场景**

| 场景                 | 记忆层参与      | 效果                     |
| -------------------- | --------------- | ------------------------ |
| 首次项目配置         | L3读取USER.md   | 自动选择用户偏好的技术栈 |
| 遇到之前解决过的错误 | L2情节记忆检索  | 直接复用历史解决方案     |
| 跨会话继续开发       | L2+L3恢复上下文 | 无需重新说明项目背景     |
| 代码风格统一         | L3语义记忆      | 自动遵循用户编码规范     |

#### 3.4 MCP (Model Context Protocol) 集成模块

与V2.0版本保持一致，通过MCP协议连接外部工具和数据源，实现能力无限扩展：

- **MCP Client**：连接外部MCP Server（GitHub、Figma、Puppeteer等）
- **MCP Server**：将本系统工具暴露为MCP Server
- **多Server管理**：通过MCPHost管理多个连接，支持命名空间隔离

#### 3.5 文件系统与Git操作模块

与V2.0版本保持一致，基于genkitx-fs和simple-git实现完整的文件系统和Git操作能力。

#### 3.6 Agent实时监控模块

在V2.0监控功能基础上，新增**自进化状态监控**：

| 新增监控项         | 说明                            | 界面展示               |
| ------------------ | ------------------------------- | ---------------------- |
| **Skill生成事件**  | Agent触发Skill创建的实时通知    | 事件时间线上的特殊标记 |
| **技能库状态**     | 当前技能库的Skill数量、版本分布 | 技能库仪表盘           |
| **记忆检索命中率** | 情节记忆检索的命中情况统计      | 记忆效率报表           |
| **自进化收益**     | Skill复用节省的Token数和时间    | 进化收益卡片           |

**技术实现**：利用Genkit的OpenTelemetry集成，结合自进化子系统的事件总线，通过Electron IPC实时推送到渲染进程。

### 八、附录：核心术语表

| 术语                     | 解释                                                         |
| ------------------------ | ------------------------------------------------------------ |
| **Genkit**               | Google开源AI应用开发框架，支持Node.js和Go，提供类型安全的Agent和Flow编排 |
| **Hermes Agent**         | Nous Research开源的自进化AI智能体框架，核心特性是技能闭环和持久化记忆 |
| **Skills闭环**           | 从经验提取到知识存储再到智能检索的完整链路，形成可复用、可迭代的“方法资产库” |
| **三层记忆架构**         | 工作记忆（会话级）、情节记忆（任务级）、语义记忆（知识级）的分层记忆系统 |
| **渐进式披露**           | 分两级加载Skill内容以节约Token的策略：L0只加载列表，L1加载完整详情 |
| **Agent-as-Tool**        | Genkit中通过`asTool()`将Agent封装为可被其他Agent调用的工具   |
| **MCP**                  | Model Context Protocol，AI应用与外部工具/数据源的标准连接协议 |
| **Orchestrator型架构**   | 以中心网关为核心的多Agent调度架构                            |
| **Learning Agent型架构** | 以自我进化为核心的单Agent架构，强调记忆和技能积累            |
